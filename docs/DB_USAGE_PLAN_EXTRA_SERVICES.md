## 2. Использование таблицы `discounts`

**Статус:** Таблица содержит смешанные данные (маркетинговые преимущества и расчетные скидки), которые не выводятся на фронтенде.

**План использования:**
1.  **Marketing (UI):** Записи с типом `bonus` (например, "Бесплатная диагностика") использовать для вывода блока преимуществ (USP) в шапке сайта или карточке товара.
2.  **Logic (Calc):** Записи с типом `percentage` (например, "-10% на второе устройство") зарезервировать для логики калькулятора корзины.

## 3. Логика поля `service_type` (Main vs Extra)

Поле необходимо для фильтрации услуг в прайс-листе.

| Тип | Описание | Примеры | Где выводить |
| :--- | :--- | :--- | :--- |
| **MAIN** | Основной технический ремонт, требующий разборки устройства. | Замена дисплея, АКБ, пайка, **стекло камеры**. | Основной список прайс-листа. |
| **EXTRA** | Сопутствующие услуги, аксессуары, программный ремонт. | Поклейка пленки, чистка, перенос данных. | Блок "Рекомендуем" (Cross-sell) или отдельная секция. |
